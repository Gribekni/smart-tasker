# Функциональные требования

## Ввод данных
- Пользователь может ввести произвольный текст
- Текст может содержать несколько мыслей или действий

## Обработка текста
- Система должна анализировать введённый текст
- Из текста должны выделяться конкретные задачи
- Каждая задача должна быть короткой и чёткой
- Задачи должны формулироваться в повелительном виде (например: "Скачать FL Studio")

## Отображение
- Список задач отображается сразу после обработки
- Каждая задача отображается отдельным пунктом

## Ограничения
- Не требуется регистрация пользователя
- Не требуется сложная бизнес-логика
- Проект ориентирован на простоту
